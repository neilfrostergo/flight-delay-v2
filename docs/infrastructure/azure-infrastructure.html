🏗️ 1. Overview

The flight-delay-v2 platform runs as a containerised Node.js/Express/PostgreSQL application on Azure Container Apps, fronted by Azure Front Door Premium in production. Two environments share the same subscription and container registry.

🟢 Production
Resource Groupfdv2-prod-rg
Domain*.delayedpaid.co.uk
NetworkPrivate (Front Door + Private Link)
WAFYes — Premium SKU
🟣 UAT
Resource Groupfdv2-uat-rg
Domain*.uat.delayedpaid.co.uk
NetworkPublic ACA environment
WAFNo

Key architectural properties

  • All secrets are stored in per-environment Azure Key Vaults; no credentials are baked into container images.
  • Container Apps pull images from fdv2acr.azurecr.io using user-assigned managed identities (no registry passwords).
  • The production Container App environment is internal (no public IP), accessible only through Azure Front Door via Private Link.
  • PostgreSQL servers are fully private (delegated subnet, private DNS zone, no public endpoint).
  • Event Hubs receives OAG flight alert events over a private endpoint.

⚠️ 14. Gaps & Items Requiring Attention

Security

Hardcoded credentials in diagnostic jobs
fdv2-migrate-diag and fdv2-seed-diag have plaintext database credentials and admin seed passwords embedded in their container environment config. These jobs should be deleted or converted to use Key Vault references (as fdv2-migrate-prod does).
Storage account TLS version
fdv2prodsa has minimum TLS set to 1.0. Should be raised to 1.2 to align with the Event Hub namespace.
No private endpoint on ACR
fdv2acr uses public network access. Container Apps pull images over the public internet.
GitHub Actions OIDC lacks scoped role assignment
The OIDC service principal has no explicit Contributor role on the resource groups — deployments rely on the subscription Owner role inherited from the human admin account. A dedicated least-privilege role should be assigned.

Infrastructure Gaps

No UAT Event Hub
UAT uses the 30-second DB poller instead of Event Hub. The Event Hub consumer code path is never validated before reaching production.
No UAT Front Door / WAF
UAT container app environment has public network access enabled with no WAF in front.
No geo-redundant database backups
Neither environment has geo-redundant backup enabled. A regional Azure outage would require restoring from the most recent local backup.
Azure Communication Services not integrated
fdv2-comms and fdv2-email exist but the application still uses SMTP.
ACR retention policy disabled
Images are not automatically purged. Tag count will grow indefinitely.
No automated migration step in CI/CD
DB migrations must be triggered manually after each deployment.
Two orphaned Log Analytics workspaces in UAT
workspace-fdv2uatrgLbuD and workspace-fdv2uatrg3Mav are leftovers from earlier failed UAT provisioning and can be deleted.
Snowflake credentials for UAT sync job unclear
The UAT fdv2-sync-job is triggered by the GitHub Actions workflow but no UAT-specific Snowflake credentials are visible — it may be sharing production Snowflake credentials.
